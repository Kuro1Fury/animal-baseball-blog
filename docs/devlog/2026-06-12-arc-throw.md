## 旧传球 vs 新传球

| | 旧（`throw_completed`）| 新（弧线 + `throw_landed`）|
|---|---|---|
| 飞行路径 | 直线匀速到目标点 | 斜抛：水平匀速 + 竖直重力 |
| 是否落地 | 瞬间到达，状态 → DEAD | 落地弹跳 → GroundRolling |
| 接球判定 | 到达即完成 | 飞行中随时可以接（catch in flight）|
| 方向跟踪 | 动态追踪 target fielder | 出手锁定方向 |

## ball_controller.gd：斜抛实现

### 新增 @export 参数

```gdscript
@export var throw_start_vz:          float = 130.0  # 出手纵向初速度
@export var throw_max_bounces:       int   = 2       # 最多弹跳次数
@export var throw_bounce_vz_damping: float = 0.50   # 弹跳后竖直速度保留
@export var throw_bounce_h_damping:  float = 0.80   # 弹跳后水平速度保留
@export var throw_bounce_min_vz:     float = 55.0   # 低于此速度直接转地滚
```

### `_tick_throwing()` — 每帧积分

```gdscript
func _tick_throwing(delta: float) -> void:
    plane_pos         += _throw_velocity * delta    # 水平匀速
    vertical_velocity -= GRAVITY * delta            # 竖直减速（重力）
    height_z          += vertical_velocity * delta  # 高度更新

    if height_z <= GROUND_BALL_HEIGHT_THRESHOLD and vertical_velocity < 0.0:
        height_z = 0.0
        if _bounce_count < throw_max_bounces and abs(vertical_velocity) >= throw_bounce_min_vz:
            _bounce_count    += 1
            vertical_velocity = -vertical_velocity * throw_bounce_vz_damping  # 反弹
            _throw_velocity  *= throw_bounce_h_damping                         # 水平减速
        else:
            # 弹跳次数用完或速度太低 → 转地滚
            hit_velocity      = _throw_velocity
            state             = BallState.GROUND_ROLLING
            throw_landed.emit()
```

这里用的是**手动欧拉积分**（不借助 Godot 的物理引擎）：每帧对速度积分得到位移，对加速度积分得到速度。参见 [godot-notes: 手动斜抛物理](/godot-notes/projectile-physics) 了解为什么不用 RigidBody2D。

### 出手方向锁定

```gdscript
func start_throw(from: Vector2, to: Vector2, speed: float, ...) -> void:
    _throw_velocity = (to - from).normalized() * speed  # 出手时锁定
```

旧版每帧动态追踪 target fielder（守备员走动会影响球的方向）；新版出手后方向固定，更符合现实。

## 信号变化：`throw_completed` → `throw_landed`

```gdscript
# 旧（已废弃，保留 API 兼容但不再 emit）
signal throw_completed

# 新
signal throw_landed   # THROWING → GroundRolling，表示球落地
```

GameDirector 连接新信号：

```gdscript
ball.throw_landed.connect(_on_throw_landed)
```

`_on_throw_landed` 处理漏接的情况——球落地后如果跑者已到垒，判 Safe；若 ball 在守备员接球范围内，触发接球判定。

## 飞行中接球：`_on_throw_caught_in_flight`

弧线传球期间，守备员在球的飞行路径上触发接球：

```gdscript
# game_director.gd
func _on_throw_caught_in_flight() -> void:
    # 接球成功 → 走 Force Out / Double Play 链路
    # 与落地接球逻辑相同，传球链路完整保留
```

这是 Arc Throw 的核心好处之一：守备员不用等球落地，飞行中就能接，然后继续传球。

## 同批修复的 4 个 Bug

| Bug | 原因 | 修复 |
|-----|------|------|
| play 结束后球瞬移 | waiting 时球还在飞 | `_enter_waiting_for_next_play` 时立即隐藏球 |
| DEAD 状态按 Space 捡不起来 | match 缺少 DEAD 分支 | 补充 `DEAD` case |
| 传球漏接时跑者不判 Safe | `_on_throw_landed` 缺结算 | 补充 runner Safe 处理 |
| Force Out at 1B 后传不出球 | deactivate 过早 | 对称 `_resolve_force_play_2b` 的 continue-play 逻辑 |

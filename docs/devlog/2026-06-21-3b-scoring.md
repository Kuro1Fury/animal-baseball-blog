## 强迫推进链（Force Chain）

击球时，满垒场景下三条推进同时启动：

```gdscript
func _do_swing() -> void:
    _batter_runner_to_1b_result = "Pending"
    if first_base_occupied:
        _runner_1b_advancing    = true
        _runner_1b_to_2b_result = "Pending"
        if second_base_occupied:
            _runner_2b_advancing     = true
            _runner_2b_to_3b_result  = "Pending"
            if third_base_occupied:
                _runner_3b_advancing      = true
                _runner_3b_to_home_result = "Pending"
```

四条 result 变量（`_runner_3b_to_home_result` / `_runner_2b_to_3b_result` / `_runner_1b_to_2b_result` / `_batter_runner_to_1b_result`）扩展了原有的 Multi-Result 屏障：`_check_play_results_complete()` 需要**所有 Pending 都结算**才进入 waiting。

## `_build_multi_result_text()`：四行结果文案

```gdscript
func _build_multi_result_text() -> String:
    var parts: Array[String] = []
    if _runner_3b_to_home_result == "Scored": parts.append("Scored!")
    if _runner_2b_to_3b_result == "Out":  parts.append("Force Out at 3B!")
    elif _runner_2b_to_3b_result == "Safe": parts.append("Runner Safe at 3B!")
    if _runner_1b_to_2b_result == "Out":  parts.append("Force Out at 2B!")
    elif _runner_1b_to_2b_result == "Safe": parts.append("Runner Safe at 2B!")
    if _batter_runner_to_1b_result == "Out":  parts.append("Force Out at 1B!")
    elif _batter_runner_to_1b_result == "Safe": parts.append("Safe!")
    var text := "\n".join(parts) if not parts.is_empty() else "Out!"
    if _runner_1b_to_2b_result == "Out" and _batter_runner_to_1b_result == "Out":
        return "Double Play!\n" + text
    return text
```

满垒场景三杀示例：`Triple Play!\nForce Out at 3B!\nForce Out at 2B!\nForce Out at 1B!`

### `_has_pending_force_at_3b()` — Force 解除逻辑

不是所有场景下 Runner2B 的推进都是强迫的：

```gdscript
func _has_pending_force_at_3b() -> bool:
    # 只有当后面的跑者（Runner1B、BatterRunner）都没被封杀，
    # Runner2B 才被强迫推进（前者出局 → Runner2B 可以停在三垒）
    return _runner_2b_to_3b_result == "Pending" \
        and _runner_1b_to_2b_result != "Out" \
        and _batter_runner_to_1b_result != "Out"
```

同样的逻辑也适用于 `_has_pending_force_at_2b()`——这是棒球规则"强迫跑垒"的本质：只要后方垒位必须有人上来，前方跑者就被迫跑。

## Force Chain Continuation：踩完一个垒继续传

踩三垒封杀后，如果 Runner1B 或 BatterRunner 还是 Pending，持球继续：

```gdscript
func _resolve_force_play_3b() -> void:
    _runner_2b_to_3b_result = "Out"
    _add_runner_out()
    _check_play_results_complete()

    if not _is_result_locked() \
            and (_runner_1b_to_2b_result == "Pending" or _batter_runner_to_1b_result == "Pending") \
            and _active_fielder != null:
        # 重新进入持球状态 → 玩家继续传向 2B 或 1B
        ball.catch()
        _fielder_holding_ball      = true
        _active_fielder.controlled = true
```

踩三垒 → 可继续传二垒 → 可继续传一垒：理论上可以完成 Triple Play。

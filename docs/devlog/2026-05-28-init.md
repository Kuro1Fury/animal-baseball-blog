## 关键决策

根据当前仓库代码推断，初期规划确定了以下方向：

- **引擎**：Godot 4（GDScript）
- **视角**：2D 俯视角，用 z 变量模拟高度（而不是真 3D）
- **核心玩法**：投球 → 挥棒 → 守备 的完整棒球回合循环
- **角色**：5 种动物，各有不同数值参数

### 为什么不用真 3D？

2D + 逻辑 z 变量的方案权衡了复杂性：

- 渲染顺序、调试可视化、碰撞判定都在 2D 里更简单
- z 高度只影响视觉效果和接球椭圆判定
- 不需要 3D 物理引擎

详见 [z 高度模拟](/godot-notes/z-height-simulation)。

# 2026-06-07 — 传二垒封杀（Force Out at Second）

> Commit：`6502f5c`（共 1 个）

## 做了什么

`6502f5c` — Force Out at Second v0.1 — 传球到二垒后，玩家控制持球守备员走到二垒脚踩封杀正在推进的 Runner1B。这是继 Force Out at 1B（路径 A/B）之后的第三条封杀路径。

## 三条封杀路径

到这个 commit 为止，`game_director.gd` 的 `_process()` 里有三条封杀路径：

| 路径 | 触发条件 | 封杀谁 |
|------|----------|--------|
| A（Hold-and-Stamp at 1B）| 持球守备员走到一垒 | 打者跑者 |
| A2（Hold-and-Stamp at 2B）| 持球守备员走到二垒 | Runner1B |
| B（Pending Force at 1B）| 一垒守备员自动走到垒 | 打者跑者 |

路径 A2 的判定条件：

```gdscript
# 路径 A2：传球到 2B 后玩家走到二垒，仍可封杀推进中的 Runner1B
if _fielder_holding_ball and _active_fielder != null \
        and _throw_target_position == "2B" and _runner_1b_advancing:
    if _active_fielder.global_position.distance_to(_second_base_point.global_position) <= force_base_touch_radius:
        _do_fielder_stamp_second()
        return
```

注意：此时还有 `_throw_target_position == "2B"` 的限制（后来在 `f9f8a0e` 里去掉了）。

## 状态跟踪变量

新增 `_last_force_2b_state: String`，取值为 `"None" / "Pending" / "Safe" / "Out"`：

```gdscript
var _last_force_2b_state: String = "None"
```

状态流转：
- 击球时一垒有人 → `"Pending"`
- Runner1B 自然到达二垒 → `"Safe"`（在 `_on_runner_1b_reached_second()` 里设置）
- 持球守备员踩二垒封杀 → `"Out"`（在 `_resolve_force_play_2b()` 里设置）

这个变量的作用是让调试标签可以同时显示两个封杀状态，便于观察 play 进展：

```gdscript
var reason_line := "Reason: %s  Force1B: %s  Force2B: %s" % [
    reason_text, _last_force_play_state, _last_force_2b_state
]
```

## 封杀函数

```gdscript
func _do_fielder_stamp_second() -> void:
    result_label.visible  = false
    _fielder_holding_ball = false
    ball.visible          = true
    if _active_fielder != null and _active_fielder.has_method("flash_catch"):
        _active_fielder.flash_catch()
    _deactivate_fielder()
    _resolve_force_play_2b()

func _resolve_force_play_2b() -> void:
    _last_force_2b_state = "Out"
    _runner_1b_advancing = false
    _runner_1b.visible   = false
    first_base_occupied  = false
    first_base_runner    = null
    _play_sfx("out")
    _register_out("Force Out at 2B!")
```

结构与 `_do_fielder_stamp_first()` + `_resolve_force_play()` 完全对称——这个命名和分层模式在后续迭代里会反复出现。

## 此时的局限性

- `_throw_target_position == "2B"` 限制：只有主动传二垒的持球者才能踩，SS 位置的守备员即使持球走到二垒也不触发
- 单结果 play：Force Out at 2B 后，打者跑者到达一垒的结果不会被显示（后来由 Multi-Result 解决）

## Hold-and-Stamp 流程（`83aaacd`）

```
接球 → ball.catch() → ball.visible = false
     → _fielder_holding_ball = true
     → UI 显示持球提示
玩家移动持球者 →
     每帧检测距离一垒 ≤ force_base_touch_radius
     → _do_fielder_stamp_first()
     → _resolve_force_play()
```

`ball.visible = false` 让球消失，视觉上表示球在守备员手里。

## Numpad 传球的控制权转移（`a50797b`）

```gdscript
func _start_throw_to_fielder(source, target) -> void:
    ball.start_throw(from, tgt_local, throw_to_first_speed)
    source.controlled = false    # 玩家不再控制发球者
    target.controlled = true     # 玩家立刻控制接球者
    _active_fielder   = target
```

传球发起后，玩家可以立刻移动接球者去接球位置，不用等球飞到。

## Line2D Bug 修复（`c7b233e`）

> "将动态创建的 Line2D ring 改为场景静态节点，修复圆圈不显示问题"

最初跑者圆圈用 `Line2D.new()` 在运行时动态创建，但圆圈不显示。改为在编辑器里预置 `Line2D` 节点，运行时只更新 `points` 属性：

```gdscript
# 之前：Line2D.new() → 不显示
# 之后：预置节点 + 只更新 points
var rr_pts := PackedVector2Array()
for i in 25:
    var angle := TAU * i / 24.0
    rr_pts.append(Vector2(cos(angle), sin(angle)) * 28.0)
_runner_ring.points = rr_pts
```

经验：可视化调试元素优先用预置场景节点，不要动态 `new()`。

## 一垒→二垒推进（`8f6aae1`）

```gdscript
# 击球时，如果一垒有人：
if first_base_occupied:
    if not second_base_occupied:
        first_base_occupied  = false
        _runner_1b_advancing = true  # 启动推进动画
```

每帧推进：

```gdscript
func _advance_runner_1b(delta: float) -> void:
    var to_2b := _second_base_point.global_position - _runner_1b.global_position
    if to_2b.length() <= 12.0:
        _on_runner_1b_reached_second()
        return
    _runner_1b.global_position += to_2b.normalized() * runner_advance_speed * delta
```

**MVP 限制**：二垒有人时推进被阻止，无回垒规则。
